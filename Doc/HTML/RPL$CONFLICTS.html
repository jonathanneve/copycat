RPL$CONFLICTS
Database tables | See Also
Show All

The RPL$CONFLICTS table is a log of all the currently conflicting data changes in the database. It has the following fields: 

 

Field name 
Data type 
Description 
CODE
 
Integer 
Integer code uniquely identifying each conflict. 
USER1
 
String
 
Replication nodes involved in the conflict. 
USER2
 
String
 
 
CHOSEN_USER
 
String
 
Replication node chosen as having the correct version of the record. By default, this field is blank. 
CONFLICT_DATE
 
Date/Time 
Date and time when the conflict was first recorded. 
TABLE_NAME
 
String
 
Name of the table. 
REMOTE_TABLE_NAME 
String 
Name of the table in the remote database 
PRIMARY_KEY_VALUES
 
String
 
Primary key values. 
REF_VALUE
 
String 
Deprecated -- don't use for now. 
SQL1 
Memo 
Names and values of the fields on node 1 
SQL2 
Memo 
Names and values of the fields on node 2 
CHANGED_FIELDS1 
Memo 
List of fields changed by node 1 
CHANGED_FIELDS2 
Memo 
List of fields changed by node 2
