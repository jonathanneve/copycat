RPL$LOG
Database tables | See Also
Show All

The RPL$LOG table is a log of every data change that takes place in the database. One line is created per user in RPL$USERS. It has the following fields: 

 

Field name 
Type 
Description 

TABLE_NAME

 

String

 
Table name 

PRIMARY_KEY_VALUES

 

String

 
Values of the primary key fields 

PRIMARY_KEY_SYNC

 

String

 
Synchronization statements for each of the primary key fields These are SQL statements that can be used for fetching unique primary key values from the remote server. 

UNIQUE_KEY_SYNC

 

String

 
Synchronization statements for each of the unique key fields. These are SQL statements that can be used for fetching unique field values from the remote server. 

PROCEDURE_STATEMENT

 

String

 
SQL of the EXECUTE PROCEDURE statement that is to be executed in order to replicate a stored procedure. 

LOGIN

 

String

 
Replication node that this line is addressed to. 

OPERATION_DATE

 

Date/Time

 
Date and time of the data change. This field is merely informational, and is not used internally by CopyCat. 

CONFLICT

 

String

 
'Y' or 'N', to indicate whether a conflict has been logged for this line or not. 

CODE

 

Integer

 
Integer code uniquely identifying the log line. 
SENT_FROM 
String 
Replication node that this line originates from. 
OPERATION_TYPE 
String 
I, U or D to indicate whether it was an insert, an update or a delete 
CHANGE_NUMBER 
Integer 
Unique number identifying the change
